# Модульная структура сервера

## Структура проекта

```
app/
├── __init__.py
├── main.py                    # Главное приложение FastAPI
├── core/                      # Основная бизнес-логика
│   ├── __init__.py
│   ├── models.py              # Модели данных
│   ├── client_manager.py      # Менеджер клиентов
│   ├── command_handler.py     # Обработчик команд
│   └── websocket_handler.py  # WebSocket обработчик
├── routes/                     # REST API роуты
│   ├── __init__.py
│   ├── clients.py            # Роуты для управления клиентами
│   └── commands.py           # Роуты для управления командами
├── schemas/                   # Pydantic схемы
│   └── cmd.py                # Схемы команд
└── utils/                     # Утилиты
    └── encryption.py          # Функции шифрования
```

## Основные компоненты

### Core модули

- **`models.py`** - Pydantic модели для типизации данных
- **`client_manager.py`** - Управление подключениями клиентов
- **`command_handler.py`** - Обработка команд и результатов
- **`websocket_handler.py`** - WebSocket соединения и шифрование

### Routes модули

- **`clients.py`** - REST API для управления клиентами
- **`commands.py`** - REST API для отправки команд

### Utils модули

- **`encryption.py`** - E2E шифрование (AES-GCM + HMAC)

## Запуск

### Старый способ (unified_server.py)
```bash
python unified_server.py
```

### Новый модульный способ
```bash
python run_server.py
# или
python -m app.main
```

## Преимущества модульной структуры

1. **Разделение ответственности** - каждый модуль отвечает за свою область
2. **Легкость тестирования** - можно тестировать модули изолированно
3. **Масштабируемость** - легко добавлять новые функции
4. **Читаемость** - код лучше организован и понятен
5. **Переиспользование** - модули можно использовать в других проектах

## Миграция

Старый `unified_server.py` теперь просто импортирует и запускает модульное приложение из `app.main`. Вся функциональность сохранена, но код лучше организован.
